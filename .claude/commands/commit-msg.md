# コミットメッセージ提案

以下の手順でコミットメッセージを提案してください：

1. `git status` で変更状態を確認
2. `git diff --stat` で変更ファイルの概要を確認
3. `git diff` で詳細な変更内容を確認（大きい場合は主要ファイルのみ）
4. `git log --oneline -10` で直近のコミットスタイルを確認

## 分析ポイント
- 変更の種類（feat, fix, docs, refactor, test, chore）
- 影響範囲（どのモジュール/機能に影響するか）
- 変更の目的（なぜこの変更が必要か）

## 出力形式
- 変更内容のまとめ（箇条書き）
- コミットメッセージの提案（2-3パターン）
  - シンプル版
  - 詳細版（本文付き）
- 直近のコミットスタイルに合わせる
