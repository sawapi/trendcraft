# Implementation Plan Generator

以下の手順で実装計画を作成してください：

## 手順

1. 現在のプロジェクト状態と、`plans/` ディレクトリに既存の計画ファイルがあるか確認
2. 既存の計画がある場合：進捗を要約し、次のフェーズを特定
3. 新規計画の場合：関連コードを探索し、変更が必要な全ファイルを特定

## 計画のフォーマット

番号付きフェーズで構成し、各フェーズに以下を含める：
- **Scope**: 作成/変更するファイル一覧
- **Tasks**: 具体的な作業項目
- **Acceptance Criteria**: ビルド通過、テスト通過など
- **Complexity**: S / M / L

## ルール

- 計画は `plans/<feature-name>.md` に保存
- コード内のコメントやラベルは英語で記述
- ユーザーにどのフェーズから実装するか確認してから着手
- 1フェーズの変更は管理可能な範囲に収める（目安: 5ファイル以内）
