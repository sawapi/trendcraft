## 移動平均線

### どんな指標？

移動平均線は、一定期間の価格の平均を計算することで価格データを滑らかにします。短期的なノイズを除去し、全体的なトレンドの方向を把握するのに役立ちます。

### 種類

| 種類 | 説明 | 特徴 |
|------|------|------|
| **SMA**（単純移動平均） | すべての価格に同じ重み | 滑らか、反応が遅い |
| **EMA**（指数移動平均） | 直近の価格に重みを置く | 新しいデータに敏感 |
| **WMA**（加重移動平均） | 直近に線形的に重みを増加 | EMAとSMAの中間的な感度 |

### 見方

```
価格がMAより上 → 上昇トレンド（強気）
価格がMAより下 → 下降トレンド（弱気）
```

### よく使われる期間

| 期間 | 用途 |
|------|------|
| 5-10 | 短期トレンド |
| 20-25 | 中期トレンド |
| 50-75 | 中長期トレンド |
| 200 | 長期トレンド |

### ゴールデンクロス / デッドクロス

2本の移動平均線が交差する時の強力なシグナルです：

```
ゴールデンクロス: 短期MAが長期MAを上抜け → 買いシグナル
デッドクロス:     短期MAが長期MAを下抜け → 売りシグナル
```

**例:** 5日MAが25日MAを上抜ける

### ポイント

- 複数のMAを組み合わせて使う（例: 5日、25日、75日）
- 期間が長いほど、シグナルの重要性が高い
- トレンド相場で有効、レンジ相場では信頼性が低下

## RSI（相対力指数）

### どんな指標？

RSIは価格変動の速度と大きさを0〜100のスケールで測定します。買われすぎ・売られすぎの状態を判断するのに役立ちます。

### 見方

```
RSI > 70  → 買われすぎ（価格が高すぎる可能性）
RSI < 30  → 売られすぎ（価格が安すぎる可能性）
RSI = 50  → 中立
```

### 主要シグナル

| 状態 | 解釈 | 検討するアクション |
|------|------|-------------------|
| RSIが70超え | 買われすぎゾーン | 売り・利益確定を検討 |
| RSIが30割れ | 売られすぎゾーン | 買いを検討 |
| RSIが50を上抜け | 強気モメンタム | トレンド確認 |
| RSIが50を下抜け | 弱気モメンタム | トレンド確認 |

### RSIダイバージェンス

ダイバージェンスは、価格とRSIが逆方向に動く現象です：

```
強気ダイバージェンス:
  価格: 安値更新  +  RSI: 安値切り上げ  →  上昇反転の可能性

弱気ダイバージェンス:
  価格: 高値更新  +  RSI: 高値切り下げ  →  下落反転の可能性
```

**重要:** ダイバージェンスは反転の*可能性*を示すものであり、確定ではありません。必ず確認を待ちましょう。

### ポイント

- RSIが70を超えてもすぐに売らない（強いトレンドは買われすぎが続く）
- ダイバージェンス＋他の確認シグナルを探す
- レンジ相場で最も有効

## MACD

### どんな指標？

MACD（移動平均収束拡散法）は、2本の移動平均線の関係を示します。トレンドの方向、勢い、反転の可能性を把握するのに役立ちます。

### 構成要素

```
MACDライン   = 12期間EMA - 26期間EMA
シグナルライン = MACDラインの9期間EMA
ヒストグラム  = MACDライン - シグナルライン
```

### 見方

#### 1. ゼロライン

```
MACDがゼロより上 → 強気トレンド（短期MA > 長期MA）
MACDがゼロより下 → 弱気トレンド（短期MA < 長期MA）
```

#### 2. シグナルラインとの交差

```
MACDがシグナルを上抜け → 買いシグナル（強気）
MACDがシグナルを下抜け → 売りシグナル（弱気）
```

#### 3. ヒストグラム

```
ヒストグラム拡大（プラス） → 強気モメンタム増加
ヒストグラム縮小          → モメンタム弱まり
ヒストグラムがマイナス    → 弱気モメンタム
```

### 視覚的ガイド

```
        MACDライン
           ↓
     ~~~~~/\~~~~~  ← シグナルライン
         /  \
    ____/    \____

   ▓▓▓▓          ← ヒストグラム（ゼロ以上＝強気）
   ▓▓▓▓▓▓
   ▓▓▓▓▓▓▓▓
   ▓▓▓▓▓▓
───────────────── ゼロライン
         ░░░░░░
         ░░░░░░░░ ← ヒストグラム（ゼロ以下＝弱気）
         ░░░░░░
```

### MACDダイバージェンス

RSIダイバージェンスと同様：

```
強気: 価格が安値更新、MACDは安値切り上げ
弱気: 価格が高値更新、MACDは高値切り下げ
```

### ポイント

- MACDは遅行指標（トレンドを確認するが、予測はしない）
- トレンド相場で最も有効
- 他の指標と組み合わせて精度を上げる

## ストキャスティクス

### どんな指標？

ストキャスティクスは、現在の価格が一定期間の高値-安値レンジのどこに位置するかを測定します。0〜100の範囲で推移します。

### 種類

| 種類 | 計算方法 | 最適な用途 |
|------|----------|-----------|
| **ファスト** | 生の%Kと平滑化%D | 素早いシグナル（ノイズ多め） |
| **スロー** | 平滑化された%Kと%D | 信頼性の高いシグナル（ノイズ少なめ） |

### 構成要素

```
%Kライン = メインライン（速い）
%Dライン = シグナルライン（%Kの3期間MA、遅い）
```

### 見方

```
80以上 → 買われすぎゾーン
20以下 → 売られすぎゾーン
```

### 主要シグナル

| シグナル | 条件 | 意味 |
|----------|------|------|
| 買い | %Kが%Dを上抜け（売られすぎゾーン<20で） | 強気反転 |
| 売り | %Kが%Dを下抜け（買われすぎゾーン>80で） | 弱気反転 |

### ポイント

- スローストキャスティクスが一般的（だましが少ない）
- RSI同様、強いトレンドでは買われすぎ/売られすぎが続く
- レンジ・横ばい相場で最も有効

## ボリンジャーバンド

### どんな指標？

ボリンジャーバンドは、価格を包む3本のラインで構成されます。市場のボラティリティに応じて拡大・縮小します。

### 構成要素

```
上バンド  = 中央バンド + (2 × 標準偏差)
中央バンド = 20期間SMA
下バンド  = 中央バンド - (2 × 標準偏差)
```

### 見方

#### バンド幅 = ボラティリティ

```
バンドが狭まる（スクイーズ） → ボラティリティ低下、ブレイクアウト近い
バンドが広がる              → ボラティリティ上昇
```

#### 価格の位置

```
価格が上バンドにタッチ → 買われすぎの可能性
価格が下バンドにタッチ → 売られすぎの可能性
価格が中央バンドで反発 → 中央バンドがサポート/レジスタンスとして機能
```

### %Bインジケーター

%Bは価格がバンド内のどこに位置するかを示します：

```
%B > 1.0  → 価格が上バンドより上
%B = 0.5  → 価格が中央バンド付近
%B < 0.0  → 価格が下バンドより下
```

### スクイーズシグナル（SQ）

スクイーズはボリンジャーバンドで最も重要なシグナルの一つです：

```
スクイーズ = バンド幅が直近の下位5〜10%に縮小
           = 「嵐の前の静けさ」
           = ボラティリティ縮小は大きな動きの前兆であることが多い
```

**SQを見たら:**
1. ブレイクアウトに備える
2. 価格がどちらの方向に抜けるか注視
3. 出来高の確認でシグナルを強化

### ポイント

- ボリンジャーバンドは方向を予測しない、ボラティリティのみ
- スクイーズ＋出来高急増＝強いシグナル
- トレンド指標（MA、MACD）と組み合わせて方向を判断

## DMI/ADX

### どんな指標？

DMI（方向性指数）はトレンドの方向と強さを測定します。ADXは方向に関係なくトレンドの強さを測定します。

### 構成要素

```
+DI = プラス方向性指標（買い圧力）
-DI = マイナス方向性指標（売り圧力）
ADX = 平均方向性指数（トレンドの強さ）
```

### ADXの見方

```
ADX < 20  → トレンド弱い / 横ばい相場
ADX 20-25 → トレンド発生中
ADX > 25  → 強いトレンド
ADX > 50  → 非常に強いトレンド
```

### DIの見方

```
+DI > -DI → 上昇トレンド（買い手優勢）
-DI > +DI → 下降トレンド（売り手優勢）
```

### 主要シグナル

| シグナル | 条件 | 意味 |
|----------|------|------|
| 買い | +DIが-DIを上抜け + ADX > 25 | 強い上昇トレンド開始 |
| 売り | -DIが+DIを上抜け + ADX > 25 | 強い下降トレンド開始 |

### ポイント

- ADXは方向を示さない、強さのみ
- ADXが低い環境でのDIクロスオーバーは信頼性が低い
- ADX上昇 = トレンド強化（トレンドフォローに好適）
- ADX低下 = トレンド弱化（利益確定を検討）

## 一目均衡表

### どんな指標？

一目均衡表は日本で開発された包括的なテクニカル指標で、トレンド方向、サポート/レジスタンス、モメンタムを一目で把握できます。

### 構成要素

```
転換線（Tenkan-sen）  = 9期間の(最高値 + 最安値) / 2  → 短期トレンド
基準線（Kijun-sen）   = 26期間の(最高値 + 最安値) / 2 → 中期トレンド
先行スパンA          = (転換線 + 基準線) / 2 を26期間先に表示
先行スパンB          = 52期間の(最高値 + 最安値) / 2 を26期間先に表示
遅行スパン           = 終値を26期間過去に表示
```

### 雲（Kumo）

先行スパンAとBの間の領域が「雲」と呼ばれます：

```
価格が雲の上 → 上昇トレンド（強気）
価格が雲の下 → 下降トレンド（弱気）
価格が雲の中 → トレンド不明確、レンジ相場
雲が厚い     → 強いサポート/レジスタンス
雲が薄い     → 弱いサポート/レジスタンス
```

### 主要シグナル

| シグナル | 条件 | 意味 |
|----------|------|------|
| 強い買い | 価格が雲を上抜け + 転換線 > 基準線 | 上昇トレンド開始 |
| 強い売り | 価格が雲を下抜け + 転換線 < 基準線 | 下降トレンド開始 |
| 三役好転 | 転換線>基準線 + 価格>雲 + 遅行スパン>価格 | 最強の買いシグナル |
| 三役逆転 | 転換線<基準線 + 価格<雲 + 遅行スパン<価格 | 最強の売りシグナル |

### ポイント

- 日本株では9, 26, 52が標準設定
- 雲は26期間先を見ているため、将来のサポート/レジスタンスを予測可能
- 複数の要素が揃うほど信頼性が高い

## スーパートレンド

### どんな指標？

スーパートレンドはATR（真の値幅）を使ってトレンドの方向と動的なサポート/レジスタンスを計算する指標です。トレーリングストップとしても使えます。

### 構成要素

```
上バンド = HL2 + (倍率 × ATR)
下バンド = HL2 - (倍率 × ATR)

価格が上バンドを上抜け → 強気トレンド、下バンドがサポートになる
価格が下バンドを下抜け → 弱気トレンド、上バンドがレジスタンスになる
```

### 見方

```
direction = 1  → 強気トレンド（緑）、スーパートレンド = サポート
direction = -1 → 弱気トレンド（赤）、スーパートレンド = レジスタンス
```

### 主要シグナル

| シグナル | 条件 | 意味 |
|----------|------|------|
| 買い | direction が -1 から 1 に変化 | 上昇トレンド転換 |
| 売り | direction が 1 から -1 に変化 | 下降トレンド転換 |

### ポイント

- 期間と倍率で感度を調整（デフォルト: 10期間、倍率3）
- 倍率が小さいほど感度が高い（だましも多い）
- トレーリングストップとして使用すると効果的

## CCI（コモディティチャネルインデックス）

### どんな指標？

CCIは価格が平均からどれだけ乖離しているかを測定します。元は商品市場向けですが、株式にも広く使われています。

### 計算方法

```
CCI = (標準価格 - 標準価格のSMA) / (0.015 × 平均偏差)
標準価格 = (高値 + 安値 + 終値) / 3
```

### 見方

```
CCI > +100 → 買われすぎ（強気モメンタム）
CCI < -100 → 売られすぎ（弱気モメンタム）
CCI が 0 を上抜け → 強気転換
CCI が 0 を下抜け → 弱気転換
```

### 主要シグナル

| シグナル | 条件 | 意味 |
|----------|------|------|
| 買い | CCI が -100 を上抜け | 売られすぎからの反発 |
| 売り | CCI が +100 を下抜け | 買われすぎからの反落 |

### ポイント

- RSIと異なり、±100 を超えることがある（強いトレンド）
- ダイバージェンスも有効
- レンジ相場で最も信頼性が高い

## ウィリアムズ%R

### どんな指標？

ウィリアムズ%Rはストキャスティクスの逆バージョンで、-100〜0の範囲で推移します。買われすぎ・売られすぎを判断します。

### 計算方法

```
%R = (最高値 - 終値) / (最高値 - 最安値) × -100
```

### 見方

```
%R > -20 → 買われすぎゾーン
%R < -80 → 売られすぎゾーン
%R = -50 → 中立
```

### 主要シグナル

| シグナル | 条件 | 意味 |
|----------|------|------|
| 買い | %R が -80 を上抜け | 売られすぎからの反発 |
| 売り | %R が -20 を下抜け | 買われすぎからの反落 |

### ポイント

- ストキャスティクスと同様の解釈
- 短期トレードに有効（デフォルト14期間）
- 強いトレンドでは極端な値が続くことがある

## ROC（変化率）

### どんな指標？

ROC（Rate of Change）は、現在の価格が過去n期間の価格からどれだけ変化したかをパーセントで表します。モメンタム指標の中で最もシンプルです。

### 計算方法

```
ROC = ((現在価格 - n期間前の価格) / n期間前の価格) × 100
```

### 見方

```
ROC > 0 → 価格上昇中（強気モメンタム）
ROC < 0 → 価格下落中（弱気モメンタム）
ROC が 0 を上抜け → 強気転換
ROC が 0 を下抜け → 弱気転換
```

### 主要シグナル

| シグナル | 条件 | 意味 |
|----------|------|------|
| 買い | ROC がゼロラインを上抜け | 上昇モメンタム開始 |
| 売り | ROC がゼロラインを下抜け | 下落モメンタム開始 |

### ポイント

- 期間を短くするとより敏感になる（デフォルト12期間）
- ダイバージェンスも有効
- 他の指標と組み合わせて使うのがベスト

## ピボットポイント

### どんな指標？

ピボットポイントは前日の高値・安値・終値から、当日のサポートとレジスタンスのレベルを計算します。デイトレーダーに人気の指標です。

### 計算方式

| 方式 | 特徴 |
|------|------|
| **Standard** | 最も一般的、(H+L+C)/3 |
| **Fibonacci** | フィボナッチ比率を使用 |
| **Woodie** | 当日の始値を重視 |
| **Camarilla** | 終値ベース、狭いレンジ |
| **DeMark** | 価格の位置関係で計算を変える |

### 見方

```
ピボット（P） = (前日高値 + 前日安値 + 前日終値) / 3

サポートレベル: S1, S2, S3（下に向かって強くなる）
レジスタンスレベル: R1, R2, R3（上に向かって強くなる）
```

### 主要シグナル

| 状況 | 解釈 |
|------|------|
| 価格 > ピボット | 強気バイアス |
| 価格 < ピボット | 弱気バイアス |
| S1/R1 で反発 | サポート/レジスタンスとして機能 |
| S1/R1 をブレイク | 次のレベル（S2/R2）を目指す |

### ポイント

- デイトレードで最も有効
- 日足ベースで計算するのが一般的
- 他の指標と組み合わせて確認

## VWAP

### どんな指標？

VWAP（出来高加重平均価格）は、出来高で重み付けした平均価格です。機関投資家が「良い価格で買えたか」の基準として使います。

### 計算方法

```
VWAP = 累計(標準価格 × 出来高) / 累計(出来高)
標準価格 = (高値 + 安値 + 終値) / 3
```

### 見方

```
価格 > VWAP → 強気（買い手優勢）
価格 < VWAP → 弱気（売り手優勢）
価格がVWAPに近づく → 平均的な価格帯に回帰
```

### 主要シグナル

| シグナル | 条件 | 意味 |
|----------|------|------|
| 買い | 価格がVWAPを上抜け | 強気転換 |
| 売り | 価格がVWAPを下抜け | 弱気転換 |
| 買い | 価格がVWAPにタッチして反発（上昇トレンド中） | サポートとして機能 |
| 売り | 価格がVWAPにタッチして反落（下降トレンド中） | レジスタンスとして機能 |

### ポイント

- 日中リセット（セッションVWAP）が一般的
- 機関投資家の参照価格として重要
- 標準偏差バンドと組み合わせると効果的

## 出来高指標

### OBV（オンバランスボリューム）

#### どんな指標？

OBVは出来高を使って累積的な買い圧力と売り圧力を追跡します。

```
価格上昇 → OBVに出来高を加算
価格下落 → OBVから出来高を減算
```

#### 見方

```
OBVと価格が共に上昇 → 健全な上昇トレンド（出来高が確認）
OBVと価格が共に下落 → 健全な下降トレンド（出来高が確認）
OBV上昇、価格横ばい → 買い集め（強気）
OBV下落、価格横ばい → 売り抜け（弱気）
```

#### OBVダイバージェンス

```
強気: 価格が安値更新、OBVは安値切り上げ
     → 価格下落にもかかわらず買い圧力が蓄積

弱気: 価格が高値更新、OBVは高値切り下げ
     → 価格上昇にもかかわらず売り圧力が蓄積
```

### MFI（マネーフローインデックス）

#### どんな指標？

MFIは出来高を組み込んだRSIのようなものです。「出来高加重RSI」とも呼ばれます。

#### 見方

```
MFI > 80 → 買われすぎ（高出来高を伴う）
MFI < 20 → 売られすぎ（高出来高を伴う）
```

MFIは出来高の大きな動きに重みを付けるため、重要な価格変動の確認に有効です。

### ポイント

- 出来高は価格の動きを確認する（価格上昇＋出来高増＝強い）
- 価格と出来高指標の乖離は反転の前兆であることが多い
- 出来高が少ない上昇は怪しい
- 出来高が多い下落はセリングクライマックス（底値の可能性）

## 高度な出来高分析

TrendCraftは基本的なOBVやMFI以外にも高度な出来高分析ツールを提供しています。

### 出来高異常検出

#### どんな指標？

出来高異常検出は、機関投資家の活動、ニュース反応、トレンド転換などの重要な市場イベントを示唆する異常な出来高スパイクを識別します。

#### 見方

```
ratio > 2.0  → 高出来高（平均の2倍）
ratio > 3.0  → 極端な出来高（平均の3倍）
zScore > 2   → 統計的に有意なスパイク
```

#### 主要シグナル

| 状況 | 意味 |
|------|------|
| 高出来高 + 価格ブレイクアウト | 確認されたブレイクアウト |
| サポートでの極端な出来高 | セリングクライマックス/底値の可能性 |
| レジスタンスでの極端な出来高 | 売り抜け/天井の可能性 |
| 出来高急増 + 価格変動なし | 買い集め/売り抜け |

### Volume Profile

#### どんな指標？

Volume Profileは、一定期間の各価格帯での出来高分布を示します。最も取引が行われた価格帯（サポート/レジスタンスレベル）を明らかにします。

#### 主要構成要素

```
POC（Point of Control）= 最大出来高の価格帯（最強のS/R）
VAH（Value Area High） = 70%出来高の上限
VAL（Value Area Low）  = 70%出来高の下限
Value Area             = 総出来高の70%を含む価格帯
```

#### 見方

```
価格がPOC付近    → 強いサポート/レジスタンスゾーン
価格がValue Area内 → フェアバリューゾーン
価格がVAHより上   → オーバーシュートの可能性
価格がVALより下   → アンダーバリューの可能性
```

#### トレード戦略

| 戦略 | エントリー | ターゲット |
|------|-----------|-----------|
| 平均回帰 | 価格がVAH/VALにタッチ | POCへの回帰 |
| ブレイクアウト | 価格がVAHを出来高増で上抜け | 次のレジスタンス |
| サポートトレード | 価格がPOCで反発 | 直近高値 |

### 出来高トレンド確認

#### どんな指標？

出来高トレンド確認は、出来高が現在の価格トレンドを支持しているかを分析します。健全なトレンドには確認する出来高が伴います。

#### 見方

```
isConfirmed = true   → 出来高が価格方向を確認
hasDivergence = true → 出来高が価格と乖離（警告）
confidence > 70      → 高い信頼度のシグナル
```

#### 主要パターン

| パターン | 意味 | 示唆 |
|---------|------|------|
| 価格上昇 + 出来高増加 | 確認済み上昇トレンド | トレンド継続の可能性 |
| 価格下落 + 出来高増加 | 確認済み下降トレンド | 強い売り |
| 価格上昇 + 出来高減少 | 弱気ダイバージェンス | 上昇力弱まり |
| 価格下落 + 出来高減少 | 強気ダイバージェンス | 売り枯れ |

#### 使用例

```typescript
import { volumeAnomaly, volumeProfile, volumeTrend } from 'trendcraft';

// 出来高スパイクを検出
const anomalies = volumeAnomaly(candles, { period: 20 });
anomalies.filter(d => d.value.isAnomaly).forEach(d => {
  console.log(`${d.time}: ${d.value.level}出来高 (${d.value.ratio.toFixed(1)}倍)`);
});

// Volume Profileを分析
const profile = volumeProfile(candles, { period: 20 });
console.log(`POC: ${profile.poc}, Value Area: ${profile.val} - ${profile.vah}`);

// トレンド確認をチェック
const trends = volumeTrend(candles);
const latest = trends[trends.length - 1].value;
if (latest.hasDivergence) {
  console.log('警告: 出来高ダイバージェンス検出');
}
```

### CMF（チャイキンマネーフロー）

#### CMFとは？

CMFは、価格が高値-安値レンジ内のどこで終値をつけたかを分析し、出来高で重み付けして買い/売り圧力を測定します。値の範囲は-1〜+1です。

```
CMF > 0   → 買い圧力（蓄積）
CMF < 0   → 売り圧力（分配）
CMF > +0.1 → 強い買い圧力
CMF < -0.1 → 強い売り圧力
```

#### CMFの計算方法

```
終値位置 = (終値 - 安値) - (高値 - 終値)
           ─────────────────────────────
                  (高値 - 安値)

CMF = N期間の(終値位置 × 出来高)の合計
      ────────────────────────────────
         N期間の出来高の合計
```

- 価格が高値付近で終値：プラスの寄与
- 価格が安値付近で終値：マイナスの寄与
- 出来高がシグナルを増幅

#### 主要なシグナル

| CMFパターン | 価格動向 | 解釈 |
|-------------|----------|------|
| CMFが0を上抜け | 上昇トレンド中 | 蓄積確認 |
| CMFが0を下抜け | 下降トレンド中 | 分配確認 |
| CMFがプラス | 価格が新高値 | 強い上昇トレンド |
| CMFがマイナス | 価格が新安値 | 強い下降トレンド |
| CMFが価格と乖離 | 価格上昇、CMF下降 | 警告：トレンド弱体化 |

#### 使用例

```typescript
import { cmfAbove, cmfBelow, and, priceAboveSma } from 'trendcraft';

// エントリー：上昇トレンド中の蓄積フェーズ
const entry = and(
  cmfAbove(0),           // 買い圧力
  priceAboveSma(50),     // 50日MAより上
);

// イグジット：分配開始
const exit = cmfBelow(-0.1);  // 強い売り圧力
```

### OBV（オンバランスボリューム）

#### OBVとは？

OBV（オンバランスボリューム）は、上昇日に出来高を加算し、下落日に減算する累積指標です。資産への出来高の流入/流出を示します。

```
上昇日（終値 > 前日終値）: OBV = 前日OBV + 出来高
下落日（終値 < 前日終値）: OBV = 前日OBV - 出来高
変化なし:                   OBV = 前日OBV
```

#### 読み方

```
OBV上昇  → 買い手優勢（蓄積）
OBV下降  → 売り手優勢（分配）
OBV横ばい → 中立/レンジ相場
```

#### 主要なシグナル

| OBVパターン | 価格パターン | 意味 |
|-------------|--------------|------|
| OBV上昇 | 価格上昇 | 上昇トレンド確認 |
| OBV下降 | 価格下落 | 下降トレンド確認 |
| OBV上昇 | 価格横ばい | 隠れた蓄積（強気） |
| OBV下降 | 価格横ばい | 隠れた分配（弱気） |
| OBV横ばい | 価格上昇 | 弱い上昇トレンド（注意） |

#### OBV vs CMF：使い分け

| 指標 | 最適な用途 | 特徴 |
|------|-----------|------|
| **CMF** | 現在の買い/売り圧力 | 短期、範囲制限（-1〜+1）、明確な閾値 |
| **OBV** | 累積的な出来高フロー | 長期トレンド、範囲制限なし、ダイバージェンスに有効 |

**CMFを使う場面：**
- 閾値を使った明確な売買シグナルが必要な時
- 短期の蓄積/分配を分析する時
- 異なる銘柄間で比較したい時

**OBVを使う場面：**
- 価格とのダイバージェンスを探す時
- 長期的な出来高トレンドを分析する時
- ブレイクアウトを出来高で確認する時

#### 使用例

```typescript
import { obvRising, obvCrossUp, cmfAbove, and } from 'trendcraft';

// 強い蓄積：複数の出来高確認
const strongEntry = and(
  cmfAbove(0),         // 現在の買い圧力
  obvRising(10),       // 10日間でOBVが上昇トレンド
);

// OBVモメンタムが強気に転換
const obvBullish = obvCrossUp(5, 20);  // 短期MAが長期MAを上抜け
```

### 出来高シグナル

#### volumeAboveAverage

出来高が連続して平均を上回る期間を検知します。持続的な高活動期間の特定に有用です。

```typescript
import { volumeAboveAverage } from 'trendcraft';

const signals = volumeAboveAverage(candles, {
  period: 20,            // 20日平均
  minRatio: 1.2,         // 平均の120%以上
  minConsecutiveDays: 3  // 3日以上連続
});

signals.forEach(s => {
  console.log(`高出来高: ${s.consecutiveDays}日間、平均の${(s.ratio * 100).toFixed(0)}%`);
});
```

#### volumeAccumulation vs volumeAboveAverage

| シグナル | 方法 | 最適な用途 |
|----------|------|-----------|
| `volumeAccumulation` | 線形回帰の傾き | 加速する出来高（強まる傾向）の検出 |
| `volumeAboveAverage` | 単純な比率比較 | 持続的な高出来高の検出 |

## 相対強度（RS）

### RSとは？

相対強度（RS）は、株式のパフォーマンスをベンチマーク指数（S&P 500や日経225など）と比較します。市場をアウトパフォームまたはアンダーパフォームしている銘柄を特定するのに役立ちます。

### 基本概念

```
RS > 1.0  →  ベンチマークをアウトパフォーム
RS < 1.0  →  ベンチマークをアンダーパフォーム
RS = 1.0  →  ベンチマークと同等
```

### なぜRSを使うのか？

| メリット | 説明 |
|---------|------|
| **市場リーダー** | 市場より強い銘柄を見つける |
| **トレンド確認** | 強い銘柄は強いままになりやすい（モメンタム） |
| **リスクフィルター** | 上昇トレンド時にラガードを避ける |
| **セクターローテーション** | セクター/銘柄のリーダーシップ変化を特定 |

### マンスフィールドRS

マンスフィールドRSは、RSラインが移動平均からどれだけ離れているかを測定します：

```
Mansfield RS > 0  →  RS強まっている
Mansfield RS < 0  →  RS弱まっている
```

### 基本的な使い方

```typescript
import { benchmarkRS, isOutperforming, rankByRS } from 'trendcraft';

// ベンチマークに対するRS計算
const rs = benchmarkRS(stockCandles, nikkei225Candles, { period: 52 });

const latest = rs[rs.length - 1].value;
console.log(`RS Rating: ${latest.rsRating}`);        // パーセンタイル（0-100）
console.log(`超過リターン: ${latest.outperformance.toFixed(1)}%`);
console.log(`Mansfield RS: ${latest.mansfieldRS?.toFixed(2)}`);

// シンプルなアウトパフォームチェック
if (isOutperforming(stockCandles, benchmarkCandles, 52, 10)) {
  console.log('ベンチマークを10%以上アウトパフォーム');
}

// 複数銘柄のランキング
const rankings = rankByRS(symbolsMap, { benchmarkSymbol: 'TOPIX' });
rankings.slice(0, 5).forEach((r, i) => {
  console.log(`#${i + 1}: ${r.symbol} (RS Rating: ${r.rsRating})`);
});
```

### RSを使ったトレード戦略

| 戦略 | RS条件 | その他の条件 |
|------|--------|-------------|
| **市場リーダー** | RS Rating > 80 | 価格が上昇トレンド |
| **RSブレイクアウト** | RSが52週高値 | 出来高急増 |
| **モメンタム** | RS上昇 + 50超 | ゴールデンクロス |
| **ラガード回避** | RS Rating < 20 | -（これらの銘柄は避ける） |

### バックテストでのRS

```typescript
import { rsAbove, rsRising, rsRatingAbove, setBenchmark, and } from 'trendcraft';

// エントリー: 強いRS + トレンド確認
const entry = and(
  rsAbove(1.0),         // アウトパフォーム
  rsRising(),           // RS改善中
  rsRatingAbove(70),    // 上位30%
  goldenCross()         // テクニカルエントリー
);

// ベンチマーク付きバックテスト実行
runBacktest(candles, entry, exit, {
  capital: 1000000,
  setup: (indicators) => {
    setBenchmark(indicators, nikkei225Candles);
  }
});
```

## 価格パターン

### 価格パターンとは？

価格パターンは、反転や継続を示す認識可能なチャートフォーメーションです。市場参加者の集団心理から形成されます。

### パターンの種類

| タイプ | 方向 | シグナル |
|--------|------|---------|
| **ダブルトップ** | 弱気 | 上昇トレンド後の反転 |
| **ダブルボトム** | 強気 | 下降トレンド後の反転 |
| **ヘッドアンドショルダー** | 弱気 | 主要な反転パターン |
| **逆ヘッドアンドショルダー** | 強気 | 主要な反転パターン |
| **カップ・ウィズ・ハンドル** | 強気 | 継続パターン（William O'Neil） |

### ダブルトップ / ダブルボトム

```
ダブルトップ（M型）:          ダブルボトム（W型）:

  ピーク1   ピーク2              ボトム1   ボトム2
    ___       ___              ▼          ▼
   /   \     /   \            /          /
  /     \___/     \          /    ___   /
         ボトム    ▼ 下抜け      /   \/
                              ピーク
                              ▲ 上抜け
```

**確認条件**: 価格が中間のボトム/ピークを下抜け/上抜け。

### ヘッドアンドショルダー

```
      ヘッド
       /\
      /  \
     /    \
    /      \___
左肩           右肩
    \          /
     \        /
      \      /
   ----\----/----  ネックライン
        ▼
     下抜け
```

**ポイント**:
- 左肩と右肩は同程度のレベル
- ヘッドが最も高い（逆の場合は最も低い）
- ネックラインがボトム（逆の場合はピーク）を結ぶ
- 目標 = ネックラインからのパターン高さ

### カップ・ウィズ・ハンドル

```
  リム        リム
   │          │
   │ カップ __/│
   │/   \___/ │  ← ハンドル（小さなプルバック）
   └──────────┴──▶ ブレイクアウト

- カップ: U字型、V字型ではない
- 深さ: 通常12-35%
- ハンドル: リムからの小さなプルバック（< 12%）
```

### 基本的な使い方

```typescript
import { doubleTop, headAndShoulders, cupWithHandle } from 'trendcraft';

// パターン検出
const doubleTops = doubleTop(candles);
const headShoulders = headAndShoulders(candles);
const cups = cupWithHandle(candles);

// 確認済みパターンをチェック
doubleTops.filter(p => p.confirmed).forEach(p => {
  console.log(`ダブルトップ確認 ${new Date(p.time)}`);
  console.log(`目標: ${p.pattern.target}`);
  console.log(`信頼度: ${p.confidence}%`);
});

// カップ・ウィズ・ハンドルのトレード
cups.forEach(p => {
  if (p.confirmed && p.confidence > 70) {
    console.log('高信頼度のカップブレイクアウト！');
    console.log(`エントリー: ${p.pattern.keyPoints[2].price}`);  // 右リム
    console.log(`目標: ${p.pattern.target}`);
    console.log(`ストップ: ${p.pattern.stopLoss}`);
  }
});
```

### パターン信頼度

各パターンには以下に基づく信頼度スコア（0-100）があります：

| 要素 | 高信頼度 |
|------|---------|
| ピーク/ボトムの揃い | 同じレベルに近い |
| パターンの深さ | 最適な深さ範囲 |
| ネックラインの傾き | フラット = より信頼できる |
| 確認 | ブレイクアウト発生 |

### バックテストでのパターン

```typescript
import { patternDetected, anyBullishPattern, cupHandleDetected, and } from 'trendcraft';

// 弱気反転パターンでイグジット
const exit = patternDetected('head_shoulders');

// 出来高付き強気パターンでエントリー
const entry = and(
  anyBullishPattern({ confirmedOnly: true }),
  volumeAboveAvg()
);

// カップ・ウィズ・ハンドルのみトレード
const cupEntry = cupHandleDetected({ confirmedOnly: true });
```

## マルチタイムフレーム（MTF）分析

### MTF分析とは？

マルチタイムフレーム分析は、上位足の指標を使用して下位足のトレードをフィルターします。大局的なトレンドとエントリーを一致させるのに役立ちます。

### なぜMTFを使うのか？

```
上位足のトレンド = コンテキスト（「森」）
下位足のシグナル = エントリー（「木」）

上位足のトレンドに沿ったトレードは勝率を向上させます。
```

### 一般的なMTF戦略

| 戦略 | 上位足チェック | 下位足エントリー |
|------|---------------|-----------------|
| トレンド一致 | 週足RSI > 50 | 日足ゴールデンクロス |
| 押し目買い | 週足上昇トレンド | 日足RSI < 30 |
| ブレイクアウトフィルター | 月足ADX > 25 | 日足ブレイクアウト |

### 利用可能なMTF条件

```typescript
// RSIフィルター
weeklyRsiAbove(50)    // 週足RSI強気
weeklyRsiBelow(50)    // 週足RSI弱気

// トレンドフィルター
weeklyUptrend()       // 価格 > 週足SMA
weeklyDowntrend()     // 価格 < 週足SMA
weeklyTrendStrong()   // 週足ADX > 25

// 価格 vs MA
weeklyPriceAboveSma(20)  // 価格 > 週足20 SMA
monthlyPriceAboveSma(50) // 価格 > 月足50 SMA
```

### 使用例

```typescript
import { TrendCraft, weeklyRsiAbove, goldenCrossCondition, and } from 'trendcraft';

// 週足RSIが強気の時のみゴールデンクロスシグナルを採用
const result = TrendCraft.from(dailyCandles)
  .withMtf(['weekly'])  // 週足タイムフレームを有効化
  .strategy()
    .entry(and(
      weeklyRsiAbove(50),        // 週足RSI > 50（強気）
      goldenCrossCondition()     // 日足ゴールデンクロス
    ))
    .exit(deadCrossCondition())
  .backtest({ capital: 1000000 });

console.log(`勝率: ${result.winRate}%`);
```

### ポイント

- 常に上位足のトレンド方向にトレードする
- 週足RSI > 50 = 強気バイアス、< 50 = 弱気バイアス
- 月足トレンドが主要な方向を定義する
- MTF分析は弱いシグナルをフィルターしてオーバートレードを減らす

## レンジ相場検出

### レンジ相場とは？

レンジ相場（ボックス相場）とは、価格が一定の範囲内で上下動を繰り返し、明確なトレンドがない状態を指します。トレンドフォロー戦略はレンジ相場では機能しにくいため、レンジ相場の検出は重要です。

### 状態（State）

レンジ相場検出は以下の状態遷移で動作します：

| 状態 | 説明 |
|------|------|
| `NEUTRAL` | 初期状態。レンジでもトレンドでもない |
| `RANGE_FORMING` | レンジが形成されつつある（初期兆候） |
| `RANGE_CONFIRMED` | レンジ相場確定（複数の条件を満たす） |
| `RANGE_TIGHT` | 非常にタイトなレンジ（ブレイクアウト近い可能性） |
| `BREAKOUT_RISK_UP` | 上方向へのブレイクアウトリスク |
| `BREAKOUT_RISK_DOWN` | 下方向へのブレイクアウトリスク |
| `TRENDING` | トレンド相場（レンジではない） |

### 検出アルゴリズム

レンジ相場検出は複数の指標を組み合わせて判定します：

```
1. ADXによるトレンド強度チェック（ADX < 25 = 弱いトレンド）
2. ドンチャンチャネル幅のATR正規化（狭い = レンジ）
3. 線形回帰の傾き（水平に近い = レンジ）
4. +DI/-DIの差分（差が小さい = 方向性なし）
5. 連続するHH（Higher High）/ LL（Lower Low）のカウント
```

### トレンド判定理由（trendReason）

`TRENDING`状態の場合、その理由が`trendReason`で確認できます：

| 理由 | 説明 |
|------|------|
| `adx_high` | ADXが閾値（25）以上で強いトレンド |
| `price_movement` | 20日間で5%以上の価格変動 |
| `di_diff` | +DI/-DIの差分が10以上 |
| `slope` | 線形回帰傾きがATRの15%以上 |
| `hhll` | 連続するHH/LLが3回以上 |

### 使い方

```typescript
import { rangeBound } from 'trendcraft';

const rbData = rangeBound(candles);

rbData.forEach(({ time, value }) => {
  const { state, rangeScore, rangeHigh, rangeLow, trendReason } = value;

  switch (state) {
    case 'RANGE_CONFIRMED':
      console.log(`${time}: レンジ確定 (${rangeLow} - ${rangeHigh})`);
      break;
    case 'RANGE_TIGHT':
      console.log(`${time}: タイトレンジ - ブレイクアウト注意`);
      break;
    case 'BREAKOUT_RISK_UP':
      console.log(`${time}: 上方ブレイクアウトリスク`);
      break;
    case 'BREAKOUT_RISK_DOWN':
      console.log(`${time}: 下方ブレイクアウトリスク`);
      break;
    case 'TRENDING':
      console.log(`${time}: トレンド中 (理由: ${trendReason})`);
      break;
  }
});
```

### Range Score

`rangeScore`は0〜100のスコアで、数値が高いほどレンジ相場の特徴が強いことを示します：

```
rangeScore > 80  → 非常にタイトなレンジ（RANGE_TIGHT）
rangeScore > 60  → レンジ確定（RANGE_CONFIRMED）
rangeScore > 40  → レンジ形成中（RANGE_FORMING）
rangeScore <= 40 → トレンドまたは中立
```

### トレード戦略

#### レンジトレード戦略
- レンジ下限（`rangeLow`）で買い、上限（`rangeHigh`）で売り
- レンジ幅の50%以上の利益目標
- レンジ外に逃げた場合はストップロス

#### ブレイクアウト戦略
- `RANGE_TIGHT`または`BREAKOUT_RISK_*`で警戒態勢
- ブレイクアウト方向にエントリー
- 出来高増加で確認

### バックテスト条件

レンジ相場検出はバックテストでも使用できます：

| 条件 | 説明 |
|------|------|
| `rangeBreakout()` | レンジからのブレイクアウトでエントリー |
| `notInRange()` | レンジ相場ではない時のみエントリー（フィルター） |
| `rangeForming()` | レンジ形成中にイグジット |

```typescript
import { TrendCraft, goldenCross, rangeBreakout, notInRange } from 'trendcraft';

// ブレイクアウト戦略
const result1 = TrendCraft.from(candles)
  .strategy()
    .entry(rangeBreakout())
    .exit(deadCross())
  .backtest({ capital: 1000000 });

// レンジ相場を避けるフィルター
const result2 = TrendCraft.from(candles)
  .strategy()
    .entry(and(goldenCross(), notInRange()))
    .exit(deadCross())
  .backtest({ capital: 1000000 });
```

### ポイント

- ADXだけでは緩やかなトレンドを見逃すことがある（だから複合判定）
- `persistBars`オプションで状態維持期間を調整可能（デフォルト: 3本）
- `rangeScore`が高いほど信頼性が高い
- ブレイクアウトシグナルは出来高確認と組み合わせると効果的

## バックテスト

### バックテストとは？

バックテストは、過去のデータを使って取引戦略のパフォーマンスをシミュレーションすることです。TrendCraftにはプリセット条件を組み合わせたバックテスト機能があります。

### 基本的な使い方

```typescript
import { runBacktest, goldenCross, deadCross } from 'trendcraft';

const result = runBacktest(
  candles,
  goldenCross(5, 25),  // エントリー条件
  deadCross(5, 25),    // イグジット条件
  {
    capital: 1000000,    // 初期資金
    stopLoss: 5,         // 5%で損切り
    takeProfit: 10,      // 10%で利確
  }
);
```

### プリセット条件

| 条件 | 説明 |
|------|------|
| `goldenCross(short, long)` | 短期MAが長期MAを上抜け |
| `deadCross(short, long)` | 短期MAが長期MAを下抜け |
| `rsiBelow(threshold)` | RSIが閾値未満 |
| `rsiAbove(threshold)` | RSIが閾値超え |
| `macdCrossUp()` | MACDがシグナルを上抜け |
| `macdCrossDown()` | MACDがシグナルを下抜け |
| `bollingerBreakout('upper'/'lower')` | BBブレイクアウト |
| `validatedGoldenCross()` | だまし検出付きGC |
| `validatedDeadCross()` | だまし検出付きDC |

### 条件の組み合わせ

```typescript
import { and, or, not } from 'trendcraft';

// ゴールデンクロス AND RSI < 30
const entry = and(goldenCross(), rsiBelow(30));

// デッドクロス OR RSI > 70
const exit = or(deadCross(), rsiAbove(70));

// 買われすぎではない
const notOverbought = not(rsiAbove(70));
```

### バックテスト結果の見方

| 指標 | 意味 | 目安 |
|------|------|------|
| `totalReturnPercent` | 総リターン率 | プラスなら利益 |
| `winRate` | 勝率 | 50%以上が望ましい |
| `maxDrawdown` | 最大ドローダウン | 20%以下が安全 |
| `sharpeRatio` | シャープレシオ | 1以上が良好 |
| `profitFactor` | 利益/損失比 | 1.5以上が望ましい |
| `avgHoldingDays` | 平均保有日数 | 戦略による |

### リアルなシミュレーション

```typescript
const result = runBacktest(candles, entry, exit, {
  capital: 1000000,
  commission: 0,          // 固定手数料
  commissionRate: 0.1,    // 手数料率 0.1%
  slippage: 0.05,         // スリッページ 0.05%
  stopLoss: 5,            // ストップロス 5%
  takeProfit: 15,         // 利確 15%
  trailingStop: 3,        // トレーリングストップ 3%
  taxRate: 20.315,        // 日本の税率
});
```

### ポイント

- 過去のパフォーマンスは将来を保証しない
- 十分な期間でテストする（複数年分）
- 手数料とスリッページを必ず考慮
- 最大ドローダウンに耐えられるか確認
- だまし検出（`validatedGoldenCross`等）で精度を上げる

## シグナルスコアリング

### シグナルスコアリングとは？

シグナルスコアリングは、複数のテクニカル指標を単一のスコアに統合する手法です。単一の指標に頼るのではなく、様々なシグナルに重みを付けて組み合わせることで、より信頼性の高いエントリー/エグジット判断ができます。

### なぜスコアリングを使うのか？

```
単一指標 = だましが多い
複数指標 = 確認が取れる、精度向上

スコア = アクティブなシグナルの重み付け合計
高スコア = 複数のシグナルが一致 = 高い信頼度
```

### ScoreBuilder

TrendCraftはカスタムスコアリング戦略を構築するFluent APIを提供しています：

```typescript
import { ScoreBuilder, calculateScore } from 'trendcraft';

const config = ScoreBuilder.create()
  .addPOConfirmation(3.0)      // パーフェクトオーダー (重み: 3.0)
  .addRsiOversold(30, 2.0)     // RSI < 30 (重み: 2.0)
  .addVolumeSpike(1.5, 1.5)    // 出来高スパイク (重み: 1.5)
  .addMacdBullish(1.5)         // MACDブリッシュ
  .setThresholds(70, 50, 30)   // strong, moderate, weak
  .build();

const result = calculateScore(candles, candles.length - 1, config);
```

### 利用可能なシグナル

| カテゴリ | シグナル | 説明 |
|---------|---------|------|
| **モメンタム** | `addRsiOversold(閾値, 重み)` | RSIが閾値未満 |
| | `addRsiOverbought(閾値, 重み)` | RSIが閾値超え |
| | `addMacdBullish(重み)` | MACDブリッシュクロス |
| | `addMacdBearish(重み)` | MACDベアリッシュクロス |
| | `addStochOversold(閾値, 重み)` | ストキャスティクス売られすぎ |
| | `addStochBullishCross(閾値, 重み)` | ストキャスブリッシュクロス |
| **トレンド** | `addPerfectOrderBullish(重み)` | MAパーフェクトオーダー |
| | `addPOConfirmation(重み)` | PO+確認 |
| | `addPullbackEntry(期間, 重み)` | MAへの押し目 |
| | `addGoldenCross(短期, 長期, 重み)` | ゴールデンクロス |
| | `addPriceAboveEma(期間, 重み)` | 価格がEMAより上 |
| **出来高** | `addVolumeSpike(閾値, 重み)` | 出来高 > 閾値×平均 |
| | `addVolumeAnomaly(z閾値, 重み)` | 統計的異常 |
| | `addBullishVolumeTrend(重み)` | 出来高がトレンド確認 |
| | `addCmfPositive(閾値, 重み)` | CMFがプラス |

### スコアの解釈

```
スコア >= 70 → 強いシグナル（高い信頼度）
スコア >= 50 → 中程度のシグナル（慎重に検討）
スコア >= 30 → 弱いシグナル（追加確認を待つ）
スコア < 30  → シグナルなし（アクション推奨なし）
```

### プリセット

TrendCraftは事前構築されたスコアリング戦略を提供しています：

| プリセット | フォーカス | 最適な用途 |
|-----------|-----------|-----------|
| `momentum` | RSI、MACD、ストキャス | スイングトレード |
| `meanReversion` | 売られすぎ条件 | 押し目買い |
| `trendFollowing` | パーフェクトオーダー、出来高 | トレンドフォロー |
| `balanced` | 混合シグナル | 汎用 |

```typescript
import { getPreset, scoreAbove } from 'trendcraft';

// プリセットをバックテストで使用
const result = TrendCraft.from(candles)
  .strategy()
    .entry(scoreAbove(70, "trendFollowing"))
    .exit(deadCross())
  .backtest({ capital: 1000000 });
```

### ポイント

- プリセットから始めて、トレードスタイルに合わせてカスタマイズ
- 重みが高い = より重要なシグナル
- シグナルを多くしすぎない（5〜7個が目安）
- ライブで使用する前にバックテストで検証

## ポジションサイジング

### ポジションサイジングとは？

ポジションサイジングは、各トレードにどれだけの資金を配分するかを決定します。適切なポジションサイジングはリスク管理と長期的な生存に不可欠です。

### なぜ重要か？

```
大きすぎるポジション = 高リスク、口座破綻の可能性
小さすぎるポジション = 機会損失
最適なポジション = リスクコントロール + 成長ポテンシャル
```

### 4つの手法

TrendCraftは4つのポジションサイジング手法を提供しています：

#### 1. リスクベース

リスク許容度とストップ距離からポジションサイズを計算：

```typescript
import { riskBasedSize } from 'trendcraft';

const result = riskBasedSize({
  accountSize: 100000,     // 口座資金100万円
  entryPrice: 50,          // エントリー価格50円
  stopLossPrice: 48,       // ストップ48円（4%下）
  riskPercent: 1,          // 口座の1%をリスク（1000円）
});

// 結果: 500株（25,000円ポジション）
// ストップアウト時: 1,000円の損失（口座の1%）
```

**使い時:** 最も一般的な手法。ストップロスが定義されている戦略に適している。

#### 2. ATRベース

ATR（真の値幅）を使って動的にストップ距離を設定：

```typescript
import { atrBasedSize } from 'trendcraft';

const result = atrBasedSize({
  accountSize: 100000,
  entryPrice: 50,
  atrValue: 2.5,           // 現在のATR = 2.5円
  atrMultiplier: 2,        // ストップ = 2 × ATR = 5円
  riskPercent: 1,
});

// ストップ45円（エントリーから2×ATR下）
// ポジションサイズがボラティリティに適応
```

**使い時:** ストップ距離を市場のボラティリティに適応させたい場合。

#### 3. ケリー基準

過去の勝率に基づいて最適なポジションサイズを計算：

```typescript
import { kellySize, calculateKellyPercent } from 'trendcraft';

// 最適ケリー比率を計算
const kellyPct = calculateKellyPercent(0.6, 1.5);
// 勝率60%、ペイオフレシオ1.5 → ケリー33%

const result = kellySize({
  accountSize: 100000,
  entryPrice: 50,
  winRate: 0.6,
  winLossRatio: 1.5,
  kellyFraction: 0.5,      // ハーフケリー（より安全）
});
```

**使い時:** 信頼できる過去の統計がある場合。安全のため常にハーフまたはクォーターケリーを使用。

#### 4. 固定比率

シンプルな割合ベースの配分：

```typescript
import { fixedFractionalSize } from 'trendcraft';

const result = fixedFractionalSize({
  accountSize: 100000,
  entryPrice: 50,
  fractionPercent: 10,     // 口座の10%
});

// ポジション: 10,000円 = 200株
```

**使い時:** シンプルな分散投資。ポートフォリオ配分に適している。

### ポジションサイジング比較

| 手法 | メリット | デメリット |
|------|---------|-----------|
| **リスクベース** | トレードごとのリスクをコントロール | ストップ定義が必要 |
| **ATRベース** | ボラティリティに適応 | ATR計算が必要 |
| **ケリー** | 数学的に最適 | 正確な統計が必要 |
| **固定比率** | シンプルで予測可能 | リスクを考慮しない |

### ポイント

- **1トレードあたり1〜2%以上のリスクは取らない**（長期生存のため）
- **maxPositionPercent**でポジションサイズに上限を設定
- **ATRベース**はボラティリティの高い市場で優れている
- **ハーフケリー**はフルケリーより安全（ドローダウンが少ない）
- **固定比率**は多数のポジションに分散する場合に有効

## ATRリスク管理

### ATRリスク管理とは？

ATR（真の値幅）は市場のボラティリティを測定します。ATRベースのリスク管理は、このボラティリティ指標を使って市場状況に適応する動的なストップロスと利確レベルを設定します。

### なぜATRを使うのか？

```
固定%ストップ = ボラティリティに関係なく同じ距離
ATRベースストップ = 現在の市場ボラティリティに適応

ボラティリティが高い市場 → 広いストップ（ノイズを回避）
穏やかな市場 → タイトなストップ（利益を確保）
```

### シャンデリアイグジット

ATRを使った価格追従型トレーリングストップ：

```typescript
import { chandelierExit } from 'trendcraft';

const chandelier = chandelierExit(candles, {
  period: 22,      // ATR期間
  multiplier: 3,   // 高値/安値から3×ATR
});

chandelier.forEach(({ time, value }) => {
  console.log(`ロングストップ: ${value.longStop}`);
  console.log(`ショートストップ: ${value.shortStop}`);
});
```

**仕組み:**
- ロングストップ = 最高値 - (倍率 × ATR)
- ショートストップ = 最安値 + (倍率 × ATR)
- 価格を追従し、逆方向には動かない

### ATRベースのストップレベル

エントリー、ストップ、利確レベルを計算：

```typescript
import { calculateAtrStops } from 'trendcraft';

const levels = calculateAtrStops({
  entryPrice: 100,
  atrValue: 2.5,
  stopMultiplier: 2,        // 2×ATRでストップ
  takeProfitMultiplier: 3,  // 3×ATRで利確
  direction: 'long',
});

// 結果:
// stopPrice: 95 (100 - 2 × 2.5)
// takeProfitPrice: 107.5 (100 + 3 × 2.5)
// riskRewardRatio: 1.5
```

### バックテストでATRを使用

```typescript
import { TrendCraft, goldenCross, deadCross } from 'trendcraft';

const result = TrendCraft.from(candles)
  .strategy()
    .entry(goldenCross())
    .exit(deadCross())
  .backtest({
    capital: 1000000,
    atrRisk: {
      enabled: true,
      period: 14,            // ATR期間
      stopMultiplier: 2,     // 2×ATRストップ
      takeProfitMultiplier: 3, // 3×ATR利確
    },
  });
```

### ATR倍率ガイドライン

| 倍率 | 用途 | ストップ幅 |
|------|------|----------|
| 1.0-1.5 | アグレッシブ（タイトストップ） | 狭い |
| 2.0-2.5 | 適度（標準） | 中程度 |
| 3.0+ | 保守的（ワイドストップ） | 広い |

### ポイント

- **2×ATR**がストップの一般的な出発点
- **高い倍率** = ストップアウトは少ないが、ヒット時の損失は大きい
- **リスク:リワード比**は最低1:1.5または1:2を目指す
- **シャンデリアイグジット**はトレンドでのトレーリングストップとして優秀
- **ポジションサイジングと組み合わせ**て完全なリスク管理を実現

### 完全なリスク管理の例

```typescript
import {
  atrBasedSize,
  calculateAtrStops,
  atr
} from 'trendcraft';

// ATRを計算
const atrValues = atr(candles, { period: 14 });
const currentAtr = atrValues[atrValues.length - 1].value;

// ポジションサイズを計算
const position = atrBasedSize({
  accountSize: 100000,
  entryPrice: 50,
  atrValue: currentAtr,
  atrMultiplier: 2,
  riskPercent: 1,           // 口座の1%をリスク
});

// ストップと利確を計算
const levels = calculateAtrStops({
  entryPrice: 50,
  atrValue: currentAtr,
  stopMultiplier: 2,
  takeProfitMultiplier: 3,
  direction: 'long',
});

console.log(`${position.shares}株を50円で購入`);
console.log(`ストップ: ${levels.stopPrice}`);
console.log(`ターゲット: ${levels.takeProfitPrice}`);
console.log(`リスク/リワード: 1:${levels.riskRewardRatio.toFixed(1)}`);
```

## シグナルの解釈

### シグナルの組み合わせ

単一の指標は完璧ではありません。複数のシグナルを組み合わせて精度を上げましょう：

```
強い買いシグナル:
  ✓ ゴールデンクロス（MA）
  ✓ RSIが売られすぎから上昇
  ✓ MACD強気クロスオーバー
  ✓ 出来高増加
  ✓ ADX > 25 かつ +DI > -DI

強い売りシグナル:
  ✓ デッドクロス（MA）
  ✓ RSIが買われすぎから下落
  ✓ MACD弱気クロスオーバー
  ✓ 出来高増加
  ✓ ADX > 25 かつ -DI > +DI
```

### ダイバージェンスシグナルの理解

ダイバージェンスは強力ですが、よく誤解されるシグナルです：

| シグナル | 意味 | 信頼性 |
|----------|------|--------|
| **強気ダイバージェンス** | 価格: 安値更新、指標: 安値切り上げ | 底値の可能性 |
| **弱気ダイバージェンス** | 価格: 高値更新、指標: 高値切り下げ | 天井の可能性 |

**重要な注意点:**
- ダイバージェンス = 反転の*可能性*であり、保証ではない
- 極端な水準（買われすぎ/売られすぎ）でより有効
- 確認を待つ（実際に価格が反転すること）
- 複数のダイバージェンス = より強いシグナル

### 相場環境との関係

指標は相場環境によって異なる動きをします：

| 相場タイプ | 有効な指標 | 避けるべき |
|------------|-----------|-----------|
| **トレンド相場** | MA、MACD、ADX | RSI極端値、ストキャスティクス |
| **レンジ相場** | RSI、ストキャスティクス、ボリンジャー | MAクロス |
| **高ボラティリティ** | ATR、ボリンジャーバンド | 固定ストップロス |
| **低ボラティリティ** | スクイーズ検出 | ブレイクアウトトレード（待機） |

### よくある間違い

1. **単一指標だけで判断** - 必ず確認を取る
2. **トレンドを無視** - 強いトレンドに逆らわない
3. **買われすぎ/売られすぎで即行動** - 強いトレンドでは極端な状態が続く
4. **出来高を無視** - 出来高は価格の動きを裏付ける
5. **確認を待たない** - ダイバージェンスには追随が必要

### クイックリファレンス: 指標の使い分け

| 知りたいこと | 使う指標 |
|--------------|----------|
| 全体的なトレンドの方向 | MA（50、200）、一目均衡表 |
| 短期トレンド | MA（5、20）、MACD、スーパートレンド |
| トレンドの強さ | ADX |
| 買われすぎ/売られすぎ | RSI、ストキャスティクス、MFI、CCI、ウィリアムズ%R |
| ボラティリティ | ボリンジャーバンド、ATR |
| 出来高の確認 | OBV、MFI、VWAP、出来高トレンド |
| 出来高の異常 | 出来高異常検出 |
| 重要な価格帯 | Volume Profile（POC、VAH、VAL） |
| 反転の可能性 | ダイバージェンス（RSI、MACD、OBV、出来高） |
| ブレイクアウトの準備 | ボリンジャースクイーズ、レンジ相場検出 |
| レンジ相場 | レンジ相場検出 |
| サポート/レジスタンス | ピボットポイント、一目均衡表（雲）、レンジ境界、Volume Profile |
| モメンタム変化 | ROC、MACD |
| 上位足のコンテキスト | MTF条件（週足RSI、SMA、ADX） |
| 戦略の検証 | バックテスト |

## ボラティリティレジーム

### どんな指標？

ボラティリティレジーム分類は、現在の市場のボラティリティ環境を特定し、それに応じてトレード戦略を調整するのに役立ちます。ATRパーセンタイルとボリンジャーバンド幅パーセンタイルを組み合わせて、市場を4つのレジームに分類します。

### レジームの種類

| レジーム | パーセンタイル | 特徴 |
|--------|------------|-----------------|
| `low` | ≤ 25 | 静かな相場、狭いレンジ、平均回帰戦略向き |
| `normal` | 25-75 | 平均的なボラティリティ、ほとんどの戦略が機能 |
| `high` | 75-95 | 高ボラティリティ、ストップを広げる |
| `extreme` | ≥ 95 | 非常に高いボラティリティ、ポジションサイズを縮小 |

### 読み方

```
regime = 'low'      → レンジ相場または平均回帰戦略を検討
regime = 'normal'   → 通常の戦略パラメータ
regime = 'high'     → ストップを広げ、ポジションサイズを縮小
regime = 'extreme'  → 非常に慎重に、様子見も検討
```

### トレードへの応用

```typescript
import { regimeIs, regimeNot, atrPercentAbove, and, goldenCross, bollingerTouch } from 'trendcraft';

// 低ボラティリティでのレンジ相場戦略
const rangeEntry = and(
  regimeIs('low'),
  bollingerTouch('lower')
);

// トレンド戦略で高ボラティリティを避ける
const trendEntry = and(
  regimeNot('extreme'),
  goldenCross()
);

// トレンドフォロー用にATR%でフィルタ（ボラタイルな銘柄のみ）
const volatileStocks = and(
  atrPercentAbove(2.3),  // ボラタイルな銘柄のみ
  perfectOrderBullish()
);
```

### ATR%フィルタリング

ATR%（価格に対するATRの割合）は、ボラティリティレベルで銘柄をスクリーニングするのに便利です：

```
ATR% < 1.5%   → 低ボラティリティ（債券、公益株など）
ATR% 1.5-2.3% → 中程度のボラティリティ
ATR% > 2.3%   → 高ボラティリティ（トレンドフォロー向き）
ATR% > 3%     → 非常にボラタイル（テック株、グロース株など）
```

### ヒント

- **低ボラティリティ**: レンジ相場または平均回帰戦略を検討
- **高/極端なボラティリティ**: ストップを広げ、ポジションを小さく
- **ATR%**: ボラティリティレベルで銘柄をフィルタ（トレンドフォローには通常2.3%以上が良い）
- **レジーム変化**: ボラティリティの拡大/縮小はブレイクアウトシグナルの可能性

## 戦略最適化

### どんな機能？

戦略最適化は、体系的なテストを通じてトレード戦略の最適なパラメータと条件を見つけるのに役立ちます。TrendCraftは3つの最適化方法を提供しています。

### グリッドサーチ

グリッドサーチは、パラメータ値のすべての組み合わせをテストして最適な設定を見つけます。

```typescript
import { gridSearch, param, constraint, goldenCross, deadCross } from 'trendcraft';

const result = gridSearch(
  candles,
  (params) => ({
    entry: goldenCross(params.short, params.long),
    exit: deadCross(params.short, params.long),
  }),
  [
    param('short', [5, 10, 15, 20]),
    param('long', [25, 50, 75, 100]),
  ],
  {
    metric: 'sharpeRatio',
    constraints: [
      constraint('winRate', '>=', 40),
      constraint('maxDrawdown', '<=', 30),
    ],
    topN: 5
  }
);

console.log('最適パラメータ:', result.results[0].parameters);
console.log('シャープレシオ:', result.results[0].metrics.sharpeRatio);
```

### ウォークフォワード分析

ウォークフォワード分析は、最適化されたパラメータがアウトオブサンプルデータでも機能するかを検証します。これはオーバーフィッティングを避けるのに役立ちます。

**仕組み:**
1. データをイン・サンプル（訓練）とアウト・オブ・サンプル（テスト）期間に分割
2. イン・サンプルデータでパラメータを最適化
3. アウト・オブ・サンプルデータでテスト
4. 複数の期間で繰り返し

```typescript
import { walkForwardAnalysis, param } from 'trendcraft';

const result = walkForwardAnalysis(
  candles,
  strategyFactory,
  paramRanges,
  {
    inSampleRatio: 0.7,    // 70%イン・サンプル、30%アウト・オブ・サンプル
    periods: 5,            // 5つのウォークフォワード期間
    metric: 'sharpeRatio',
  }
);
```

### 組み合わせ探索

条件プールからベストなエントリー/イグジット条件の組み合わせを見つけます。

```typescript
import { combinationSearch, createEntryConditionPool, createExitConditionPool } from 'trendcraft';

const result = combinationSearch(
  candles,
  createEntryConditionPool(),
  createExitConditionPool(),
  { metric: 'sharpeRatio', topN: 20 }
);
```

### ヒント

- **シンプルに始める**: 一度に2-3パラメータのみ最適化、すべてを同時にやらない
- **制約を使う**: 非現実的な結果をフィルタ（例: 最大ドローダウン > 50%）
- **検証する**: オーバーフィッティングをチェックするためにウォークフォワード分析を必ず使う
- **懐疑的に**: 過去のパフォーマンスは将来を保証しない

## 分割エントリー戦略

### どんな機能？

分割エントリー（スケールドエントリー）は、一度に全ポジションを建てる代わりに、資金を複数のトランシェに分割します。これにより平均エントリー価格を改善し、タイミングリスクを軽減できます。

### 戦略

| 戦略 | 説明 | 例（3トランシェ） |
|----------|-------------|---------------------|
| `equal` | 各トランシェ均等配分 | 33%, 33%, 33% |
| `pyramid` | 早いトランシェに大きい配分 | 50%, 33%, 17% |
| `reverse-pyramid` | 後のトランシェに大きい配分 | 17%, 33%, 50% |

### インターバルタイプ

| タイプ | 説明 |
|------|-------------|
| `signal` | 各エントリーシグナルでトランシェ追加 |
| `price` | priceInterval % の価格下落でトランシェ追加 |

### 使用例

```typescript
import { runBacktestScaled, goldenCross, deadCross } from 'trendcraft';

// 3トランシェでエントリー、2%下落で追加
const result = runBacktestScaled(candles, goldenCross(), deadCross(), {
  capital: 1000000,
  scaledEntry: {
    tranches: 3,
    strategy: 'pyramid',      // 50%, 33%, 17%
    intervalType: 'price',
    priceInterval: -2,        // 2%下落でトランシェ追加
  },
});

// シグナルベース: 各ゴールデンクロスで追加
const result2 = runBacktestScaled(candles, goldenCross(), deadCross(), {
  capital: 1000000,
  scaledEntry: {
    tranches: 3,
    strategy: 'equal',
    intervalType: 'signal',
  },
});
```

### 戦略の選び方

| 戦略 | 適した状況 |
|----------|----------|
| **Pyramid** | 最初のシグナルに自信がある、早めに大きいエクスポージャーが欲しい |
| **Equal** | 中立的アプローチ、バランスの取れたリスク |
| **Reverse-pyramid** | タイミングに自信がない、ナンピン買いしたい |

### ヒント

- **Pyramid**は最初のシグナルに自信があるときに良い
- **Reverse-pyramid**はナンピン買いに良い
- **価格ベースのインターバル**はボラタイルな相場で効果的
- **シグナルベースのインターバル**はトレンドフォロー戦略で効果的
- ポジションが構築されるにつれて利益を確保するために部分利確を検討

## まとめ

テクニカル分析は未来を予測するものではなく、確率を理解しリスクを管理するためのものです。TrendCraftの指標を使って：

1. **トレンドを把握する**（MA、MACD、ADX、一目均衡表、スーパートレンド）
2. **レンジ相場を検出する**（レンジ相場検出）
3. **エントリー/エグジットポイントを見つける**（RSI、ストキャスティクス、CCI、ウィリアムズ%R、クロスオーバー）
4. **出来高で確認する**（OBV、MFI、VWAP）
5. **高度な出来高分析**（出来高異常検出、Volume Profile、出来高トレンド）
6. **ボラティリティを評価する**（ボリンジャーバンド、ATR）
7. **反転の可能性を検出する**（ダイバージェンス、スクイーズ、ROC）
8. **サポート/レジスタンスを特定する**（ピボットポイント、一目均衡表の雲、レンジ境界、Volume Profile POC）
9. **マルチタイムフレーム分析**（週足/月足RSI、SMA、トレンドフィルター）
10. **戦略をバックテストする**（runBacktest + プリセット条件）

覚えておいてください：完璧な指標はありません。複数のツールを組み合わせ、常にリスク管理を行い、失っても困らない金額以上は投資しないでください。バックテストで戦略を検証し、だまし検出機能を活用して精度を高めましょう。
